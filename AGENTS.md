Перед внесением изменений:

1. Проанализируй существующую архитектуру.

2. Объясни:

- что будет изменено;
- почему это необходимо;
- какие файлы будут затронуты;
- как изменения повлияют на проект.

После внесения изменений сообщай:

- требуется ли git commit;
- требуется ли git push;
- требуется ли новый деплой Railway.
